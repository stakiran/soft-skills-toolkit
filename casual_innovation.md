# カジュアル・イノベーション
- イノベーションと出会う確率を上げること、特に「上げることができる」と考えること
- 本ツールキットでは、そのための原則や発想を整備する
- 本ツールキットの傾向
    - 個人が深く集中してゼロイチを出すことにフォーカス
    - 一方で、少人数で密にコラボレーションするタイプのツールも扱うことがある（🧑‍🤝‍🧑を付与）

# Tools
日常的に揃えたい 3つの S:

```markdown
- Slack
    - 1: 時間的な余裕が 1 日 3 時間以上ありますか？
    - 2: 時間的な余裕だけでなく、2 時間以上の発散ワークを行える「認知資源的な余裕」もありますか？
    - 3: 1+2 を数カ月以上確保できますか？
- Solo
    - 4: Slack の間は、あらゆる生物からの干渉をゼロにできますか？
    - 5: Slack の間は、他の作業や気晴らしに気が散るのをゼロにできますか？
    - 6: 休憩と休息を行う仕組みがありますか？
- Stress
    - 7: 心身を損ねることなく、そのテーマに数カ月以上持続的に取り組めますか？
    - 8: モチベーションを損ねることなく、そのテーマに数カ月以上持続的に取り組めますか？
    - 9: キャリアや直近の生活を損ねることなく、そのテーマに数カ月以上持続的に取り組めますか？
```

環境に適用するべき 4 つの S:

```markdown
- Speedy
    - スペック、レスポンス、手続きなどあらゆる面で「速い」こと
    - アスリートやプロゲーマーのように 0.x 秒でも速さを求め、それに近づいていけること
- Seamless
    - 自分にとって自然なリズムで動けること。「滑らかである」こと
    - 日々の思考や作業はもちろん、ワークとライフの切り替えも含むこと
- Silent
    - 静かであること
    - 最低でも 40 db 台以下の静音であること
- Solitary
    - 孤独であること
    - 人やペットといった生物から通知全般まで、割り込みと誘惑をゼロに近づけられること
```

# 参考: トランスジェクト
👉️ [Project vs Transject - DEV Community](https://dev.to/stakiran/project-vs-transject-cpd)

トランスジェクト(Transject)とは:

- プロジェクトに一切所属しないあり方
- 主な特徴:
    - 管理や計画とは無縁で、自律的に動く
    - 広範囲への継続的な情報発信を行う
    - 直接売上には繋がらないが、影響は与えられる

トランスジェクトをカジュアル・イノベーションとして使う場合の考え方:

- トランスジェクト・メンバーは恒常的に 3S を満たす
- しかし、本人だけではバイアスにかかるため、広範囲への情報発信を行うことでフィードバックを受け付ける

# 参考: Exploratory
~~👉️ (記事は無し)~~

Exploratory とは:

- ウォーターフォール、アジャイルに続く第三の潮流
- 「探索期間」だけ定めて、その間は何をしても良いとする
- 4 人以下の少人数で行う
- 以下は必ず用意する
    - テーマ。何について探索するかを一行で
    - 共有ワークスペース。日々考えたことやつくったものを共有するワークスペース。
- 以下は一切しない
    - 会議や会話
    - 指示や命令
    - 計画
- 以下は積極的に行う
    - 共有ワークスペースになるべく情報を置く。結果だけでなく過程も積極的に置く
    - 共有ワークスペースの内容を読んで、参考やインスピレーションにする
    - 共有ワークスペースの内容を自由に活用する
        - 遠慮やリスペクトは不要。おもちゃのように使う。好きなように使う。
