# GitHub を用いた発散ワーク
- クリエイティブ・シンキングの一フェーズである発散の作業を GitHub で行う
- n 人で 1 リポジトリを共同編集し、衝突が起きないようにテーマとコメントを書いて push や pull を行う
- ファイル名:
    - `(theme).md`
    - `(theme)-(name).md`

# Tools
テンプレート:

- <https://github.com/stakiran/divergent_work_using_github>

発散作業として本手法を使うべきかどうかを判断するチェックリスト:

```markdown
すべて Yes の場合に、使っても良い。
一つでも No があると、おそらく成立しない。

- 1: メンバー全員が GitHub および Git のスキルを持っている
    - y | n
- 2: メンバー全員が Markdown ファイルの形でコメントを書くスキルを持っている
    - y | n
- 3: Miro のようなデジタルホワイトボードによるアイデア出しに不満がある
    - y | n
- 4: 対面またはリモートで集まって、口頭でアイデアを出すやり方に不満がある
    - y | n
- 5: メンバー全員が心理的安全性を持っている、もしくは気にせず率直に意見を言える
    - y | n
- 6: アクティブに書き込むメンバーは 4 人以上、100 人以下である
    - y | n
```
