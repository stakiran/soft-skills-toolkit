# Divergent Work Using GitHub
- Run the divergence phase of creative thinking on GitHub
- Have n people co-edit a single repository; to avoid conflicts, write themes and comments and use push/pull
- File names:
    - `(theme).md`
    - `(theme)-(name).md`

# Tools
Template:
- <https://github.com/stakiran/divergent_work_using_github>

Checklist to decide whether you should use this approach for divergent work:

```markdown
It's OK to use only if everything is Yes.
If there's even one No, it probably won't work.

- 1: Everyone on the team has GitHub and Git skills
    - y | n
- 2: Everyone on the team can write comments in Markdown files
    - y | n
- 3: You're dissatisfied with ideation using a digital whiteboard like Miro
    - y | n
- 4: You're dissatisfied with generating ideas verbally when gathering in person or remotely
    - y | n
- 5: Everyone has psychological safety, or can speak candidly without worrying about it
    - y | n
- 6: The number of members who actively write is 4 or more and 100 or fewer
    - y | n
```
