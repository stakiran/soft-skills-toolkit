- [README](README.md)
- GitHub: [Repo](https://github.com/stakiran/soft-skills-toolkit), [Issues](https://github.com/stakiran/soft-skills-toolkit/issues)
- TEAラインの見方
    - T(Target): このツールの主な対象読者
    - E(Effect): 対象読者が得られる効果
    - A(Action): このツールが採用するアプローチ。対象読者にインストールされる考え方ややり方

# コミュニケーション

## > 会話と会議
- MAMA(Meeting As An AMA): [JA](mama_meeting_as_an_ama.md) [EN](mama_meeting_as_an_ama_EN.md)
    - T: メンバーからもっと意見を引き出したいマネージャー
    - E: 30 分の会議一つで 20 以上の質問に応えることができる
    - A: AMA に徹した会議を行う
- RAMA(Realtime AMA): [JA](rama_realtime_ama.md) [EN](rama_realtime_ama_EN.md)
    - T: メンバーからもっと意見を引き出したいマネージャー
    - E: メンバーの発言数を従来の 2 倍以上に増やし、かつ質問に答えるので会議体験も上げられる
    - A: 会議のアジェンダに AMA パートを加える
- RMA(Request Me Anything): [JA](rma_request_me_anything.md) [EN](rma_request_me_anything_EN.md)
    - T: 承認その他依頼を受ける立場のマネージャー
    - E: 依頼一つ一つのオーバーヘッドを最小限にできる
    - A: 典型的な依頼パターンごとに対応を最適化する（Five Category）
- 機能としてのマネージャー(Manager As A Function): [JA](manager_as_a_function.md) [EN](manager_as_a_function_EN.md)
    - T: 自身がボトルネックになっていると感じるマネージャー、またはマネージャーがボトルネックだと考えるメンバー
    - E: マネージャーのタスク処理効率が数倍以上に上がる
    - A: マネージャーは自身を機能と捉えてインタフェースを提供し、メンバーに使ってもらう

# ドキュメンテーション
- シンプレート原則: [JA](simplate_simple_template_principle.md) [EN](simplate_simple_template_principle_EN.md)
    - T: テンプレートをつくって配布する者、特にチームに適用する立場にある者
    - E: テンプレートの作成コスト、共有コスト、使用コストを従来の 1/3 にする（ここでコストとは時間）
    - A: テンプレートは一枚に収めかつカテゴリーと余白を設けて見やすくする
- サマリーを書くための TEA ライン: [JA](summary_line_TEA.md) [EN](summary_line_TEA_EN.md)
    - T: ビジネスまたはテクニカルなドキュメント・記事を書くすべての人
    - E: 読み手が価値を判断する時間コストを 1/4 にする
    - A: Target, Effect, Action の三行でサマリーを書く

# アジャイル・ワーク
- カジュアル・アジャイル: [JA](casual_ajile.md) [EN](casual_ajile_EN.md)
    - T: 計画的・管理的な仕事の仕方が染み付いており、素早く動けないマネージャーや経営者
    - E: マネジメントやコミュニケーションにかかる人件費を 1/10 にできる
    - A: アジャイルの本質を取り入れた「小さなツール」を使い分けることで、探索的に仕事をする
- Personal Daily Workspace(PDW): [JA](personal_daily_workspace.md) [EN](personal_daily_workspace_EN.md)
    - T: 仕事が場当たり的になっており、振り返りもできなくて停滞を感じているすべての人
    - E: 毎日 30 分を費やす代わりに、毎日作業と思考を記録でき、振り返りもできるようになる
    - A: 自分用のメモ領域を日ごとにつくり、そこをホームにして過ごす

# クリエイティブ・シンキング
- GitHub を用いた発散ワーク: [JA](divergent_work_using_github.md) [EN](divergent_work_using_github_EN.md)
    - T: GitHub + プレーンテキストベースで、共同でクリエイティブ・シンキングを行いたいチーム
    - E: 6 人のチームが、一つのテーマに関して、非同期的に、一週間で 10 万文字分のアイデアと議論を出せる
    - A: リポジトリを一つつくり、`(Theme).md` と `(Theme)-(MemberName).md` に従って各自書く。また push や pull もする

# メタ・ソフトスキル

## > ソフトスキル・エンジニアリング
- ソフトスキルをツール化する際に使える Seven Lists 原則: [JA](seven_lists_for_soft_skills.md) [EN](seven_lists_for_soft_skills_EN.md)
    - T: ソフトスキルの重要性を啓蒙したい者、または使いこなせるようになりたいエンジニアやマネージャー
    - E: ツールをつくる際の思考コストを減らせる。1 分で方向性を判断できるようになる
    - A: 観点が 7 つあるので、どれをつくりたいかを選ぶ
- SESSEI(Subjective Effect of Soft Skills Engineering Index): [JA](sessei_subjective_effect_of_soft_skills_engineering_index.md) [EN](sessei_subjective_effect_of_soft_skills_engineering_index_EN.md)
    - T: ソフトスキルの開発、推進、啓蒙といった「測定しづらい仕事」に従事する者
    - E: 測定可能となり、測定指標がなければ動かない人達を説得できるようになる
    - A: シンプルで答えやすい質問項目 7 つのアンケートに答えてもらう
