# Proposal Triad
- 提案の内容や表現を考えるときに使う 3 要素列挙型のフレームワーク

# Tools
死角なく理解してもらうための Why-What-How リスト

```markdown
- Q1: Why を教えて。メリット、効果、妥当性など。
- Q2: What を教えて。提案の概要やイメージなど。
- Q3: How を教えて。具体的な進め方。
```

提案を伝える前に確認するべき 3T チェックリスト:

```markdown
すべて Yes でない場合、提案はおそらく一蹴されます。

- 1: Trust、メッセージを送ったら返事してもらえる程度には信頼関係がある
    - y | n
- 2: Tradition、提案先やそこを取り巻く慣習から逸脱せずに伝えることができる
    - y | n
- 3: Theory、提案する内容は、提案先が期待する理屈に則って構成されている
    - y | n
```

企画を提案するときに確認するべきチェックリスト:

```markdown
- 1: ニーズ。必要性。お金を払ってでも欲しいと思わせる提案ができる
    - y | n
- 2: シーズ。実現性。数字と言語により論理的に仮説がつくられている。または実際に試せる実物がある
    - y | n
- 3: フィーズ（Feeds）。持続性。供給が持続的に回るだけのビジネスモデルと運用が、シーズのレベルでつくられている
```

対象読者とメリットを端的に伝えるときに使える TEA ライン:

```markdown
- Target: （誰に対してのものか）
- Effect: （Target に及ぼす効果。できれば数字で）
- Action: （Target に Effect を及ぼすために具体的に何をするか・どう捉えるか）
```
