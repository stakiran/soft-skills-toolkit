# ソフトスキルをツール化する際に使える Seven Lists 原則
- 背景:
    - ソフトスキルは難しい:
        - ハードウェアやソフトウェアの形（ハードツール）で道具化しにくい
        - 概念だけでは使いづらい
    - テンプレートやチェックリストといった形（ソフト・ツール）があると多少は嬉しい
- Seven Lists は、ソフト・ツールの主な観点を 7 つに整理したもの

# Tools
Seven Lists のリスト:

```markdown
- Task: 指示や手順を並べる。書いたとおりに從ってもらうことを想定
- Span: 時間的な長さを持つ事柄（期間）を並べる
- Trigger: 発想の参考になりそうなヒントを並べる
- Question: オープン・クエスチョンを並べる
- Check: y | n で答えられる質問（確認事項）を並べる
- Label: 分類項目を並べる。これらを使って付けてもらうことを想定
- Fill: 枠を並べる。埋めてもらうことを想定。テンプレート
```

Visual Format:

```
               Task    Span
     Trigger                  Question
            Check        Label
                   Fill
```
