# シンプレート原則
- シンプレート(Simplate - Simple Template)
- テンプレートはシンプルであるべき
- 具体的には以下すべてを満たす:
    - 読みやすさのためにカテゴリーや余白が設けられていること
    - 一枚以内に収めること
    - 補足説明は別のドキュメントに移すこと
    - 補足説明へのポインタを含む注釈も一切書かないこと

# Tools
利用するフォーマットを問う質問リスト:

```markdown
- Q1: Markdown？
- Q2: Word などリッチテキスト？また Markdown でないのはなぜですか？
- Q3: Notion や Google ドキュメントなどページ？また ファイルでないのはなぜですか？
- Q4: パワーポイントなど古典的なスライド？スライドでなければならない理由は？
- Q5: Canva や Figma など次世代的なスライド？デザインツールでなければならない理由は？
```

文書型のシンプレートをつくる際のチェックリスト

```markdown
- 1: 1ページに収まっています
    - y | n
- 2: カテゴリーと余白により読みやすさが確保されています
    - y | n
- 3: 補足説明や脚注は一切ありません
    - y | n
```
